Execution Role:
Intercepts incoming HTTP requests immediately after CSRF checks. Sanitizes spoofable headers, bypasses verification for exempted routes/methods/internal calls, validates the user JWT cookie, and attaches decoded identity to `req.user`.

Detailed Line-by-Line Logic & Code Behavior:
Line 1: Imports `jsonwebtoken`.
Lines 4-11 (`const PUBLIC_ROUTES = [...]`): Defines an array of route prefixes exempt from authentication: `/api/auth/success`, `/api/auth/github`, `/api/auth/github/callback`, `/api/health`, `/api/webhooks/github`, and `/api/internal/verify-token`.
Line 13 (`const INTERNAL_SECRET = process.env.INTERNAL_SECRET`): Loads the shared internal secret used for secure microservice communication.
Line 15 (`function auth(req, res, next)`): Defines the middleware signature.
Line 16: Logs incoming request method and URL for diagnostic tracing.
Lines 19-24 (`const hasInternalSecret = ...`): Evaluates whether the incoming request contains an `x-internal-secret` header matching the gateway's `INTERNAL_SECRET`. If verified, immediately invokes `next()` and returns, allowing trusted downstream services to bypass external JWT checks and preserve any injected identity headers.
Lines 27-28 (`delete req.headers["x-user-id"]; delete req.headers["x-user-github-id"];`): CRITICAL SECURITY SANITIZATION. Removes any incoming `x-user-id` or `x-user-github-id` headers from external requests to prevent malicious actors from spoofing identity headers before proxying.
Lines 31-33: Checks if request method is `OPTIONS` (CORS preflight request) and calls `next()` immediately to allow browser CORS negotiations to proceed.
Lines 36-38: Evaluates whether `req.originalUrl` starts with any prefix defined in `PUBLIC_ROUTES`. If matched, calls `next()` to bypass JWT verification.
Lines 41-46: Extracts the authentication token from `req.cookies.token` (populated by `cookie-parser`).

Lines 52-55: Verifies that `JWT_SECRET` is defined in environment variables. If missing, logs critical error and halts with HTTP 500 Internal Server Error.
Lines 57-65: Inside a `try...catch` block, executes `jwt.verify(token, process.env.JWT_SECRET, { issuer: "pr-tracker-auth", audience: "pr-tracker-system" })`.
  - On Success: Attaches the resulting decoded payload object to `req.user` and calls `next()` to continue pipeline execution.
  - On Failure (expired, tampered, or invalid token): Catches exception and returns HTTP 401 Unauthorized with JSON error `{"error": "Invalid token"}`.

## 1. Executive Architectural Summary & Client-Side Design Patterns

The PR Tracker frontend is structured as a Single Page Application (SPA) leveraging **React 19**, bundled and served in development via **Vite 7**, and styled utilizing **Tailwind CSS v4** alongside **Lucide React** icons.

```
+---------------------------------------------------------------------------------------+
|                                    React 19 SPA                                       |
|                                                                                       |
|   +-------------------------------------------------------------------------------+   |
|   |                              RepoContext Provider                             |   |
|   |  State: activeRepository | activePr | repos | user | refreshRepos callback   |   |
|   +-------------------------------------------------------------------------------+   |
|                                           |                                           |
|   +---------------------------------------+---------------------------------------+   |
|   |                                                                               |   |
|   v                                                                               v   |
| +------------------------------------+          +-------------------------------+ |   |
| | Public Route Container             |          | AppLayout Container (Auth)    | |   |
| | - LandingPage (Hero, Features...)  |          | - Header (Breadcrumbs, AI)    | |   |
| | - LoginPage (OAuth Entry)          |          | - Sidebar (Repo Switcher)     | |   |
| | - AuthCallback (JWT Cookie Sync)   |          | - AiSidebar (Mistral Agent)   | |   |
| +------------------------------------+          | - Outlet (Protected Pages)    | |   |
|                                                 +-------------------------------+ |   |
|                                                                 |                 |   |
|       +------------------------+------------------------+-------+--------+        |   |
|       |                        |                        |                |        |   |
|       v                        v                        v                v        |   |
| +-----------+            +-------------+         +--------------+  +------------+ |   |
| | Dashboard |            | PullRequests|         | Repositories |  | PRDetails  | |   |
| +-----------+            +-------------+         +--------------+  +------------+ |   |
+---------------------------------------------------------------------------------------+
```

### Architectural Pillars
1. **Global Gateway Targeting**: Network calls communicate uniformly across authenticated routes via an Axios client configured to target `VITE_SERVER_ENDPOINT` (pointing to the backend service router on port 5000 in development, or relative `/api` paths in containerized production).
2. **Context-Driven Workspace**: Global state is consolidated inside `RepoContext`. It manages user profiles, active workspace selections (current repository and PR), and exposes a global `refreshRepos` trigger to sync client state across disconnected component trees.
3. **Session Security**: Authentication relies on `HttpOnly` cookie-based JWTs set by the backend gateway upon OAuth redirection. Axios defaults (`withCredentials = true`) ensure credentials attach to every outgoing cross-origin API request.
4. **Resilient AI Execution Shell**: The client incorporates an AI sidebar (`AiSidebar.jsx`) connecting to a multi-agent backend via `/api/ai/agent`. The UI acts as an interactive execution shell capable of intercepting server-side tool requests (`UI_CONFIRMATION_NEEDED:`) and dispatching user permissions to trigger backend mutations.

```
+--------------------------------------------------------------------------------+
```

### `src/App.jsx`
```javascript
import { Route, Routes } from "react-router-dom";
import LandingPage from "./components/pages/LandingPage";
import AppLayout from "./components/layout/AppLayout";
import { Dashboard } from "./components/pages/Dashboard";
import { PullRequestsPage } from "./components/pages/PullRequests";
import { RepositoriesPage } from "./components/pages/Repository";
import PRDetails from "./components/pages/PRDetails";
import LoginPage from "./components/pages/Login";
import ImportReposPage from "./components/pages/ImportRepos";
import AuthCallback from "./components/pages/AuthCallback";
import axios from "axios";
import { useState, useEffect, useCallback } from "react";
import { timeAgo } from "./utils/timeAgo";
import { RepoProvider, useRepo } from "./context/RepoContext";
import { useLocation } from "react-router-dom";

const serverEndpoint = import.meta.env.VITE_SERVER_ENDPOINT || "";
const PUBLIC_ROUTES = ["/", "/login", "/auth/callback"];

function AppContent() {
  const [loading, setLoading] = useState(true);
  const [needsImport, setNeedsImport] = useState(false);
  const [bootError, setBootError] = useState(null);
  const location = useLocation();

  const {
    setRepos,
    setActiveRepository,
    activeRepository,
    setUser,
    user,
    setRefreshRepos,
  } = useRepo();

  const isPublicRoute = PUBLIC_ROUTES.some((path) => location.pathname === path);

  /* ---------- LOAD DATA ---------- */
  const loadData = useCallback(async () => {
    try {
      setBootError(null);
      const apiBase = serverEndpoint || "";

      // 1. Current user
      const userRes = await axios.get(`${apiBase}/api/db/users/me`, { withCredentials: true });
      const userData = userRes.data?.data;
      setUser(userData);

      const importedIds = userData?.repositories || [];
      const noRepos = importedIds.length === 0;
      setNeedsImport(noRepos);

      if (noRepos) {
        setRepos([]);
        setActiveRepository(null);
        return;
      }

      // 2. GitHub repos
      const repoRes = await axios.get(`${apiBase}/api/repos`, { withCredentials: true });
      const allGhRepos = repoRes.data || [];

      // 3. Filter imported
      const filtered = allGhRepos.filter((r) => importedIds.includes(r.githubRepoId));
      const mapped = filtered.map((r) => ({
        id: r.githubRepoId,
        name: r.name,
        owner: r.owner,
        fullName: r.fullName,
        prs: r.openPrs || 0,
        updated: timeAgo(r.updatedAt),
      }));

      setRepos(mapped);

      // Restore last selected repo
      const savedId = localStorage.getItem("activeRepoId");
      let toActivate = null;
      if (savedId) {
        toActivate = mapped.find((r) => String(r.id) === String(savedId));
      }
      if (!toActivate && mapped.length > 0) {
        toActivate = mapped[0];
      }
      setActiveRepository(toActivate || null);
    } catch (error) {
      console.error("Init failed", error);
      setNeedsImport(true);
      setRepos([]);
      setActiveRepository(null);
      setBootError(
        error?.response?.status === 401
          ? "Your session is not active. Please sign in again."
          : error?.message || "Failed to load your workspace."
      );
    }
  }, [setRepos, setActiveRepository, setUser]);

  /* ---------- INIT ---------- */
  useEffect(() => {
    setRefreshRepos(() => loadData);
    if (isPublicRoute) {
      setLoading(false);
      return;
    }
    setLoading(true);
    async function init() {
      await loadData();
      setLoading(false);
    }
    init();
  }, [loadData, setRefreshRepos, isPublicRoute]);

  /* ---------- LOADING & ERROR RENDERING ---------- */
  if (loading) {
    return (
      <div className="flex min-h-screen items-center justify-center bg-bg px-6">
        <div className="w-full max-w-sm rounded-2xl border border-divider bg-surface px-6 py-8 text-center shadow-lg">
          <div className="mx-auto h-10 w-10 animate-spin rounded-full border-2 border-divider border-t-accent" />
          <p className="mt-4 text-sm font-medium text-primary">Loading PR Tracker</p>
          <p className="mt-1 text-xs text-secondary">Synchronizing your session and repository data.</p>
        </div>
      </div>
    );
  }

  if (bootError && !isPublicRoute) {
    return (
      <div className="flex min-h-screen items-center justify-center bg-bg px-6">
        <div className="w-full max-w-md rounded-2xl border border-divider bg-surface p-6 text-center">
          <h1 className="text-lg font-semibold text-primary">Workspace unavailable</h1>
          <p className="mt-2 text-sm text-secondary">{bootError}</p>
          <div className="mt-5 flex flex-col gap-3 sm:flex-row">
            <button
              onClick={() => window.location.reload()}
              className="flex-1 rounded-md bg-accent px-4 py-2 text-sm font-medium text-black hover:opacity-90"
            >
              Try again
            </button>
            <a
              href="/login"
              className="flex-1 rounded-md border border-divider px-4 py-2 text-sm text-secondary hover:bg-hover hover:text-primary"
            >
              Sign in
            </a>
          </div>
        </div>
      </div>
    );
  }

  /* ---------- ROUTE REGISTRATION ---------- */
  return (
    <Routes>
      <Route path="/" element={<LandingPage />} />
      <Route path="/login" element={<LoginPage />} />
      <Route path="/auth/callback" element={<AuthCallback />} />

      <Route element={<AppLayout user={user} />}>
        <Route path="/dashboard" element={<Dashboard user={user} needsImport={needsImport} />} />
        <Route path="/pull-requests" element={<PullRequestsPage />} />
        <Route path="/repos" element={<RepositoriesPage />} />
        <Route path="/pull-requests/:id" element={<PRDetails />} />
        <Route path="/import-repos" element={<ImportReposPage />} />
      </Route>
    </Routes>
  );
}

function App() {
  return (
    <RepoProvider>
      <AppContent />
    </RepoProvider>
  );
}

export default App;
```
*   **What & Why**: The root application controller. It instantiates the global state context (`RepoProvider`), manages session verification, orchestrates initial workspace synchronization, and defines the client routing table.
*   **Async Data Flow (`loadData`)**:
    1. Queries current user profile from `/api/db/users/me`.
    2. Extracts `userData.repositories` (array of imported repository IDs). If empty, flags `needsImport = true` and halts loading.
    3. Fetches all accessible GitHub repositories from `/api/repos`.
    4. Filters GitHub repositories down to those explicitly tracked in MongoDB (`importedIds.includes(r.githubRepoId)`).
    5. Maps raw backend data into normalized UI shapes with computed relative timestamps (`timeAgo`).
    6. Checks `localStorage.getItem("activeRepoId")`. If found, sets that repo as the active workspace; otherwise defaults to the first repo in the list.
*   **State & Mutation Mechanics**:
    *   `setRefreshRepos(() => loadData)`: A critical design pattern. `loadData` is injected into the React Context state, allowing any child component in the application (e.g., modals after importing a repo) to invoke `refreshRepos()` and trigger a global workspace reload.
*   **Routing Hierarchy**:
    *   **Public Unauthenticated Routes**: `/` (Landing Page), `/login` (OAuth trigger), `/auth/callback` (OAuth callback processing).
    *   **Protected App Shell (`<AppLayout>`)**: Contains `/dashboard`, `/pull-requests`, `/repos`, `/pull-requests/:id`, and `/import-repos`.

### `src/context/RepoContext.jsx`
```javascript
import { createContext, useContext, useState, useMemo } from "react";

const RepoContext = createContext();

export function RepoProvider({ children }) {
  const [activeRepository, setActiveRepository] = useState(null);
  const [activePr, setActivePr] = useState(null);
  const [repos, setRepos] = useState([]);
  const [user, setUser] = useState(null);
  const [refreshRepos, setRefreshRepos] = useState(() => () => {});

  const contextValue = useMemo(() => ({
    activeRepository, setActiveRepository,
    activePr, setActivePr,
    repos, setRepos,
    user, setUser,
    refreshRepos, setRefreshRepos,
  }), [activeRepository, activePr, repos, user, refreshRepos]);

  return (
    <RepoContext.Provider value={contextValue}>
      {children}
    </RepoContext.Provider>
```

### `src/utils/timeAgo.js`
```javascript
export function timeAgo(dateString) {
  const diff = Date.now() - new Date(dateString).getTime();
  const minutes = Math.floor(diff / 60000);
  if (minutes < 60) return `${minutes}m ago`;

  const hours = Math.floor(minutes / 60);
  if (hours < 24) return `${hours}h ago`;

  const days = Math.floor(hours / 24);
  return `${days}d ago`;
}
```
*   **What & Why**: Pure utility function calculating relative human-readable time strings (`"12m ago"`, `"3h ago"`, `"5d ago"`) from ISO 8601 timestamps. Used across repository lists, PR tables, activity logs, and commit histories.

```javascript
import { useState, useRef, useEffect } from "react";

function RepoSwitcher({ onClose }) {
  const [open, setOpen] = useState(false);
  const ref = useRef(null);
  const { repos, activeRepository, setActiveRepository } = useRepo();

  useEffect(() => {
    function handleClick(e) { if (!ref.current?.contains(e.target)) setOpen(false); }
    document.addEventListener("mousedown", handleClick);
    return () => document.removeEventListener("mousedown", handleClick);
  }, []);

  const current = activeRepository || repos?.[0] || null;

  return (
    <div ref={ref} className="relative w-full">
      <button onClick={() => setOpen((o) => !o)} className="flex w-full items-center justify-between text-sm font-medium text-primary">
        <span className="truncate max-w-[148px]">{current?.name}</span> <ChevronDown className="ml-1 h-4 w-4 shrink-0 text-secondary" />
      </button>
      {open && (
        <div className="absolute left-0 top-8 z-50 w-52 rounded-lg border bg-surface-elev shadow-lg max-h-64 overflow-auto py-1">
          {repos.map((r) => (
            <button
              key={r.id}
              onClick={() => {
                setActiveRepository(r);
                localStorage.setItem("activeRepoId", String(r.id));
                setOpen(false);
                onClose?.();
              }}
              className={`w-full flex items-center gap-2 px-3 py-1.5 text-left text-sm hover:bg-hover ${current?.id === r.id ? "text-primary bg-selected" : "text-secondary"}`}
            >
              <FolderGit2 className="h-3.5 w-3.5 shrink-0" /> <span className="flex-1 truncate">{r.name}</span>
            </button>
          ))}
        </div>
      )}
    </div>
  );
}

function SidebarItem({ icon: Icon, label, collapsed, path, onNavigate }) {

  const active = path && (path === "/dashboard" ? location.pathname === path : location.pathname.startsWith(path));
  return (
    <Link to={path || "#"} onClick={() => onNavigate?.()} className={`flex items-center rounded-lg py-2 transition-colors ${collapsed ? "justify-center px-0" : "px-3"} ${active ? "bg-selected text-primary" : "text-secondary hover:bg-hover hover:text-primary"}`}>

    </Link>
  );
}
```

*   **What & Why**: Responsive main navigation sidebar. Houses the active repository switcher, primary view navigation, collapse state controls, and user profile management.
*   **Execution Mechanisms**:
    *   **Repository Switcher (`RepoSwitcher`)**: Renders a dropdown populated from `repos` context. Clicking an entry mutates `activeRepository` in context and persists the selection to browser `localStorage` under `activeRepoId`.
    *   **Session Termination (`handleLogout`)**: Dispatches a `POST` request to `/api/auth/logout`. The gateway invalidates the backend session and strips the HTTP-only cookie. The client then forces a full window location redirect to `/login`.
    *   **Active Route Highlight**: `SidebarItem` evaluates `location.pathname.startsWith(path)` to ensure nested routes (e.g., `/pull-requests/123`) correctly highlight the parent sidebar entry.

```javascript
function PRInsights({ prs = [] }) {
    if (!prs.length) return <div className="flex h-40 items-center justify-center rounded-2xl border bg-surface text-sm">No pull request data to analyse</div>;

    /* Status breakdown */
    const counts = { Open: 0, Draft: 0, Merged: 0, Closed: 0 };
    prs.forEach((pr) => { const s = pr.status || "Open"; counts[s] = (counts[s] || 0) + 1; });
    const total = prs.length;

    /* Top authors */
    const authorMap = {};
    prs.forEach((pr) => {
        const a = pr.author || "Unknown";
        if (!authorMap[a]) authorMap[a] = { login: a, avatar: pr.authorAvatar, open: 0, merged: 0, closed: 0 };
        if (pr.status === "Open" || pr.status === "Draft") authorMap[a].open++;
        else if (pr.status === "Merged") authorMap[a].merged++;
        else authorMap[a].closed++;
    });
    const topAuthors = Object.values(authorMap).map((a) => ({ ...a, total: a.open + a.merged + a.closed })).sort((a, b) => b.total - a.total).slice(0, 6);

    /* Average PR age */
    const openPrs = prs.filter((pr) => pr.status === "Open" || pr.status === "Draft");
    const avgAgeDays = openPrs.length ? Math.round(openPrs.reduce((sum, pr) => sum + (Date.now() - new Date(pr.created_at || Date.now()).getTime()) / 86400000, 0) / openPrs.length) : null;

    /* Recent activity (last 7 days) */
    const MS_DAY = 86400000;
    const days = Array.from({ length: 7 }, (_, i) => { const d = new Date(Date.now() - (6 - i) * MS_DAY); return { label: d.toLocaleDateString("en-US", { weekday: "short" }), date: d.toDateString(), updated: 0 }; });
    prs.forEach((pr) => { const d = new Date(pr.updated_at || "").toDateString(); const slot = days.find((s) => s.date === d); if (slot) slot.updated++; });
    const maxActivity = Math.max(...days.map((d) => d.updated), 1);

    return (
        <div className="grid grid-cols-1 lg:grid-cols-3 gap-4">
            {/* Status Breakdown Stacked Bar Card */}
            {/* Top Contributors Card */}
            {/* 7-Day Activity Histogram Card */}
        </div>
    );
}
```

### `src/components/features/repos/ImportRepoModal.jsx`
```javascript
import { useEffect, useState, useRef } from "react";
import axios from "axios";
import { Github, Search, X, CheckCircle2, Loader2 } from "lucide-react";
import { useRepo } from "../../../context/RepoContext";

const serverEndpoint = import.meta.env.VITE_SERVER_ENDPOINT || "";

export function ImportRepoModal({ open, onClose }) {
    const { user, refreshRepos } = useRepo();
    const [phase, setPhase] = useState("idle");
    const [ghRepos, setGhRepos] = useState([]);
    const [selected, setSelected] = useState([]);
    const [search, setSearch] = useState("");
    const [importing, setImporting] = useState(false);
    const [done, setDone] = useState(false);
    const [errorMsg, setErrorMsg] = useState("");
    const backdropRef = useRef(null);

    useEffect(() => {
        if (!open) return;
        setSelected([]); setDone(false); setErrorMsg(""); setPhase("loading");
        (async () => {
            try {
                const res = await axios.get(`${serverEndpoint}/api/repos`, { withCredentials: true });
                const all = res.data || [];
                const importedIds = user?.repositories || [];
                const available = all.filter((r) => !importedIds.includes(r.githubRepoId));
                setGhRepos(available);
                setPhase("idle");
            } catch (err) { setErrorMsg("Could not load repositories."); setPhase("error"); }
        })();
    }, [open, user]);

    const handleBackdrop = (e) => { if (e.target === backdropRef.current) onClose(); };
    const toggle = (id) => setSelected((p) => p.includes(id) ? p.filter((x) => x !== id) : [...p, id]);
    const toggleAll = () => setSelected(selected.length === visibleRepos.length ? [] : visibleRepos.map((r) => r.githubRepoId));

    const visibleRepos = ghRepos.filter((repo) => {
        if (!search.trim()) return true;
        const q = search.trim().toLowerCase();
        return [repo.name, repo.fullName, repo.owner].filter(Boolean).some((value) => String(value).toLowerCase().includes(q));
    });

    const handleImport = async () => {
        if (!selected.length) return;
        setImporting(true); setErrorMsg("");
        try {
            const reposToTrack = ghRepos.filter((r) => selected.includes(r.githubRepoId));
            const results = await Promise.allSettled(
                reposToTrack.map((r) => axios.post(`${serverEndpoint}/api/repos/track`, { owner: r.owner, name: r.name }, { withCredentials: true }))
            );
            const failed = results.filter((r) => r.status === "rejected");
            if (failed.length && failed.length === reposToTrack.length) throw new Error("All repo imports failed");
            setDone(true);
            await refreshRepos();
            setTimeout(() => { onClose(); setDone(false); }, 1400);
        } catch (err) { setErrorMsg("Import failed."); }
        finally { setImporting(false); }
    };

    if (!open) return null;

    return (
        <div ref={backdropRef} onClick={handleBackdrop} className="fixed inset-0 z-50 flex items-center justify-center bg-black/60">
            {/* Modal Body: Search Input, Checkbox Rows, Action Footer */}
        </div>
    );
}
```
*   **What & Why**: Modal dialog enabling users to import available GitHub repositories into their tracked workspace.
*   **Data Filtration Pattern**:
    1. On open, queries `/api/repos` to fetch all repositories accessible via the user's GitHub OAuth token.
    2. Inspects `user.repositories` from context (array of previously tracked repository IDs).
    3. Strips previously tracked repos from the list (`!importedIds.includes(r.githubRepoId)`), ensuring the modal only displays unimported targets.
*   **Concurrent Network Execution**:
    *   `handleImport` executes `Promise.allSettled` across all checked repositories, dispatching parallel `POST` requests to `/api/repos/track`. This endpoint commands the backend to save the Repository entity to MongoDB and trigger an immediate bulk sync of all its pull requests.
    *   Upon completion, invokes `refreshRepos()`, triggering a global workspace reload that updates the active sidebar switcher and repository grids.

### `src/components/features/AiSidebar.jsx`
```javascript
import { useState, useRef, useEffect } from "react";
import axios from "axios";
import { X, Send, Sparkles, Loader2 } from "lucide-react";
import { useRepo } from "../../context/RepoContext";
import ReactMarkdown from "react-markdown";
import remarkGfm from "remark-gfm";

const serverEndpoint = import.meta.env.VITE_SERVER_ENDPOINT || "";

export default function AiSidebar({ open, onClose }) {
  const [messages, setMessages] = useState([{ role: "ai", content: "Hey! I'm your PR assistant. Ask me anything..." }]);
  const [input, setInput] = useState("");
  const [loading, setLoading] = useState(false);
  const bottomRef = useRef(null);
  const inputRef = useRef(null);
  const { activeRepository, user } = useRepo();

  useEffect(() => { bottomRef.current?.scrollIntoView({ behavior: "smooth" }); }, [messages]);
  useEffect(() => { if (open) setTimeout(() => inputRef.current?.focus(), 300); }, [open]);

  const sendMessage = async () => {
    const text = input.trim();
    if (!text || loading) return;
    const userMsg = { role: "user", content: text };
    setMessages((prev) => [...prev, userMsg]); setInput(""); setLoading(true);

    try {
      const context = {};
      if (activeRepository) context.activeRepo = { id: activeRepository.id, name: activeRepository.name, owner: activeRepository.owner, fullName: activeRepository.fullName };
      if (user) context.username = user.username;

      const res = await axios.post(`${serverEndpoint}/api/ai/agent`, { query: text, context }, { withCredentials: true });
      let content = res.data?.message || "No response from AI.";
      let action = null;

      if (typeof content === "string" && content.startsWith("UI_CONFIRMATION_NEEDED:")) {
          const parts = content.split(":");
          const tool = parts[1];
          const args = parts.slice(2).join(":");
          action = { tool, args: JSON.parse(args) };
          content = `I need your permission to run **${tool}**.`;
      }

      setMessages((prev) => [...prev, { role: "ai", content, action }]);
    } catch (err) { setMessages((prev) => [...prev, { role: "ai", content: "⚠️ Something went wrong." }]); }
    finally { setLoading(false); }
  };

  const confirmAction = async (action) => {
    setMessages((prev) => [...prev, { role: "user", content: `(Confirmed execution of ${action.tool})` }]);
    setLoading(true);
    try {
      const context = {};
      if (activeRepository) context.activeRepo = { id: activeRepository.id, name: activeRepository.name, owner: activeRepository.owner, fullName: activeRepository.fullName };
      if (user) context.username = user.username;
      context.forceTool = action;

      const res = await axios.post(
        `${serverEndpoint}/api/ai/agent`,
        { query: `I confirm the execution of the tool ${action.tool} with these arguments: ${JSON.stringify(action.args)}. Please execute it now.`, context },
        { withCredentials: true }
      );
      let content = res.data?.message || "No response from AI.";
      setMessages((prev) => [...prev, { role: "ai", content }]);
    } catch (err) { setMessages((prev) => [...prev, { role: "ai", content: "⚠️ Something went wrong." }]); }
    finally { setLoading(false); }
  };

  return (
    <>
      {open && <div className="fixed inset-0 z-40 bg-black/30 lg:hidden" onClick={onClose} />}
      <aside className={`ai-sidebar ${open ? "ai-sidebar--open" : ""}`}>
        <div className="ai-sidebar__header">{/* Title & Close */}</div>
        <div className="ai-sidebar__messages">
          {messages.map((msg, idx) => (
            <div key={idx} className={`ai-msg ${msg.role === "user" ? "ai-msg--user" : "ai-msg--ai"}`}>
              <div className={`ai-msg__bubble ${msg.role === "user" ? "ai-msg__bubble--user" : "ai-msg__bubble--ai"}`}>
                {msg.role === "ai" ? (
                  <>
                    <ReactMarkdown remarkPlugins={[remarkGfm]}>{msg.content}</ReactMarkdown>
                    {msg.action && (
                      <div className="mt-3 flex gap-2">
                        <button onClick={() => confirmAction(msg.action)} className="rounded bg-green-500/20 px-3 py-1 text-xs text-green-400">Confirm</button>
                        <button onClick={() => { setMessages(p => [...p, { role: "user", content: `Cancelled ${msg.action.tool}` }]); msg.action = null; }} className="rounded bg-red-500/20 px-3 py-1 text-xs text-red-400">Cancel</button>
                      </div>
                    )}
                  </>
                ) : (<p>{msg.content}</p>)}
              </div>
            </div>
          ))}
          {loading && <Loader2 className="h-3.5 w-3.5 animate-spin" />}
          <div ref={bottomRef} />
        </div>
        <div className="ai-sidebar__input">{/* Textarea & Send button */}</div>
      </aside>
    </>
  );
}
```
*   **What & Why**: Slide-out AI assistant interface connecting directly to the backend Mistral AI agent service (`/api/ai/agent`).
*   **Context Injection Mechanism**: Every chat request enriches its payload by injecting current workspace state (`activeRepository` details and `username`). This contextualizes the backend LLM's responses without requiring explicit user prompting.
*   **Interactive Tool Execution Protocol**:
    1. If the AI agent determines a mutation or tool execution is necessary, the backend returns a special string prefixed with `UI_CONFIRMATION_NEEDED:<ToolName>:<JSONArgs>`.
    2. The client intercepts this token, parses the tool and arguments into an `action` object, and renders `"Confirm"` and `"Cancel"` action buttons beneath the chat bubble.
    3. Clicking `"Confirm"` invokes `confirmAction(action)`, sending a follow-up query to `/api/ai/agent` containing `context.forceTool = action`. This commands the backend agent to bypass confirmation checks and execute the requested tool immediately.

Purpose:
Defines the local multi-container development and testing environment. It orchestrates all 6 core microservices inside a shared virtual Docker bridge network (`backend`), ensuring strict startup dependencies and injecting local environment variable files.

Detailed Line-by-Line Logic & Code Behavior:
Lines 1-10 (mongodb service):
- `build: ./mongodb`: Instructs Docker Compose to build the container from the `mongodb/` subdirectory's Dockerfile.
- `container_name: mongodb`: Explicitly names the container `mongodb` for predictable DNS resolution within the Docker network.
- `env_file: - ./mongodb/.env`: Mounts the dedicated environment configuration file into the container runtime.
- `networks: - backend`: Attaches the MongoDB container to the isolated `backend` network. (Note: host ports are commented out, enforcing that only internal containers can connect to port 5004).

Lines 12-25 (backend service):
- `build: ./backend`: Builds the main backend microservice.
